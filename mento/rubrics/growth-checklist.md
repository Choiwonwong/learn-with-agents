# Senior Engineer Growth Checklist

## 매 세션 체크

- [ ] 오늘 작성한 코드가 있다.
- [ ] 오늘 리뷰받은 코드가 있다.
- [ ] 다음 수정 하나가 명확하다.
- [ ] 실행 또는 검증 증거가 있다.
- [ ] 공개 저장소에 안전한 내용만 남겼다.
- [ ] drift가 있으면 accept/correct/defer로 정리했다.
- [ ] 커밋/푸시가 필요한 변경인지 판단했다.
- [ ] 학습이 완전히 끝났다면 개인 지식 저장소 기록 필요 여부를 판단했다.

## 설계 사고

- [ ] 책임을 나눌 수 있다.
- [ ] tradeoff를 말할 수 있다.
- [ ] 변경에 강한 지점과 약한 지점을 구분할 수 있다.
- [ ] 추상화를 추가한 이유를 설명할 수 있다.

## 구현 습관

- [ ] 작은 단위로 실행한다.
- [ ] 에러 메시지를 읽고 가설을 세운다.
- [ ] 라이브러리 문서를 확인한다.
- [ ] reference 코드를 보기 전에 먼저 시도한다.

## 리뷰 수용

- [ ] 지적을 코드 변경으로 연결한다.
- [ ] 같은 실수를 다음 세션에서 줄인다.
- [ ] 막힌 지점을 구체적인 질문으로 바꾼다.
