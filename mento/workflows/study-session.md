# Study Session Workflow

실제 학습 세션 진행 방식입니다.

## 흐름

1. 오늘의 목표를 한 문장으로 고정한다.
2. 사고 질문 하나를 던진다.
3. 필요한 개념을 짧게 설명한다.
4. `practice/`에 직접 코드를 작성하게 한다.
5. 작은 실행 또는 검산을 한다.
6. 리뷰 기준 하나를 적용한다.
7. 다음 수정 하나를 남긴다.
8. 세션이 끝나거나 방향이 바뀌면 [Session Closeout](session-closeout.md)을 따른다.

## 이론 중심 주제 보조 흐름

기본 흐름은 위의 code-first 세션입니다. 주제가 보안, 네트워크, 운영체제, 분산 시스템처럼 이론 구조가 주 학습 대상이라면 기존 흐름을 대체하지 않고 아래 보조 흐름을 사용할 수 있습니다.

1. 오늘의 질문을 “무엇을 만들 것인가”가 아니라 “어떤 경계를 설명할 것인가”로 고정한다.
2. 위협 모델, 상태 전이, 권한 경계, 용어 차이 중 해당 주제의 중심 구조를 먼저 세운다.
3. 이론으로 예측 가능한 관찰 결과를 적는다.
4. `practice/` 코드는 구현 산출물이 아니라 관찰 도구로 작성할 수 있다.
5. 실행 결과, 브라우저 콘솔, 로그, 테스트 중 하나로 예측을 검산한다.
6. 작성한 설명을 리뷰하고, 틀린 용어/경계/원인 설명을 수정한다.
7. 다음 수정 하나를 남긴다.

## 원칙

- 설명보다 실습이 길어야 한다.
- 이론 중심 보조 흐름에서는 설명 자체가 리뷰 대상이 될 수 있다. 다만 관찰, 실험, 로그, 테스트, 다이어그램 중 하나로 설명을 검산한다.
- reference 코드는 먼저 보여주지 않는다.
- 막히면 힌트 -> 부분 예시 -> 비교 코드 순서로 제공한다.
- 매 세션은 `mento/session-log.md`에 연결된다.
- 세션 종료, 방향 변경, drift, publish 준비가 발생하면 `mento/workflows/session-closeout.md`를 따른다.
- 최신 파일 재읽기, 공개 안전, 검증, git, NexusV1 세부 gate는 closeout 문서가 canonical source다.
