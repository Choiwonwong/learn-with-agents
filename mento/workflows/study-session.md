# Study Session Workflow

실제 학습 세션 진행 방식입니다.

## 흐름

1. 오늘의 목표를 한 문장으로 고정한다.
2. 사고 질문 하나를 던진다.
3. 필요한 개념을 짧게 설명한다.
4. `practice/`에 직접 코드를 작성하게 한다.
5. 작은 실행 또는 검산을 한다.
6. 리뷰 기준 하나를 적용한다.
7. 다음 수정 하나를 남긴다.
8. 세션이 끝나거나 방향이 바뀌면 [Session Closeout](session-closeout.md)을 따른다.

## 원칙

- 설명보다 실습이 길어야 한다.
- reference 코드는 먼저 보여주지 않는다.
- 막히면 힌트 -> 부분 예시 -> 비교 코드 순서로 제공한다.
- 매 세션은 `mento/session-log.md`에 연결된다.
- 학습이 완전히 끝난 경우에만 개인 지식 저장소 기록 여부를 판단한다. 개인 저장소는 공개 저장소의 필수 절차처럼 다루지 않는다.
