# Workflows

Codex가 이 프로젝트에서 학습을 시작하고 운영하는 방식을 정의합니다.

## 핵심 워크플로우

- [Mentor Routing](../mentor-routing.md): 요청 유형별로 먼저 읽을 파일과 다음 workflow를 고르는 canonical routing surface
- [Learning Start](learning-start.md): "오늘 공부하자"가 들어왔을 때 세션을 여는 절차
- [Topic Intake](topic-intake.md): 새 주제를 모듈로 만들기 전 확인할 것
- [Mentor Challenge](mentor-challenge.md): 사용자의 요청이 흐름과 맞지 않을 때 경고/대안 제시
- [Study Session](study-session.md): 실제 학습 진행 방식
- [Code Review Session](code-review-session.md): 작성한 코드를 학습 기준으로 리뷰하는 방식
- [Session Closeout](session-closeout.md): 준비 완료, 방향 수정, drift 식별, 학습 종료를 닫는 orchestrator

## Closeout risk gates

- [Public Safety](closeout/public-safety.md): 공개 저장소 안전 검열
- [Verification](closeout/verification.md): 변경 종류별 최소 검증
- [Git Publish](closeout/git-publish.md): commit/push와 commit splitting
- [NexusV1 Record](closeout/nexus-record.md): `nexus-vault` 기반 개인 학습 기록
