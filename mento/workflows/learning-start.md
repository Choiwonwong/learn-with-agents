# Learning Start Workflow

사용자가 아래처럼 말하면 이 워크플로우를 시작합니다.

```text
오늘 공부하자
오늘 2시간 공부하고 싶어
주제: Go 동시성
Java Stream 공부하고 싶어
기존 내용 보고 다음 학습 추천해줘
이 코드 리뷰하면서 학습하고 싶어
```

## 1. 요청 분류

- `topic-specified`: 사용자가 주제를 지정함
- `recommendation-needed`: 기존 기록을 보고 추천해야 함
- `code-review-led`: 작성한 코드를 리뷰하며 학습해야 함
- `roadmap-planning`: 큰 학습 계획을 짜야 함
- `retrospective`: 회고와 다음 약점 선정이 목표임

## 2. 기존 맥락 확인

Codex는 가능한 한 먼저 아래를 확인합니다.

- `mento/roadmap.md`
- `mento/session-log.md`
- `mento/weaknesses.md`
- `mento/backlog.md`
- 관련 `learning/<track>/README.md`
- 관련 모듈의 `notes.md`

## 3. 멘토 판단

Codex는 사용자의 요청을 무조건 따르지 않습니다. 먼저 아래 중 하나를 짧게 말합니다.

- 좋은 다음 주제입니다.
- 가능하지만 선행 개념이 부족합니다.
- 지금은 이 주제보다 다른 주제를 먼저 하는 편이 낫습니다.
- 오늘 분량으로 너무 큽니다. 이 하위 목표만 다룹니다.
- 기존 약점과 연결하려면 이런 방식으로 바꾸는 편이 좋습니다.

## 4. 세션 브리프 작성

학습을 시작하기 전에 아래를 짧게 고정합니다.

- 주제
- 시간
- 선행 개념
- 오늘 만들거나 수정할 것
- 사용할 파일
- 성공 기준
- 끝날 때 남길 기록

## 5. 실습 시작

Codex는 먼저 하나의 사고 질문을 던지고, 바로 이어서 필요한 설명과 작은 구현 단계를 제공합니다.

## 6. 종료 조건

세션은 아래 세 가지가 남으면 종료할 수 있습니다.

- 작성한 코드
- 리뷰받은 코드
- 다음 수정

종료, 방향 수정, drift 식별, 커밋/푸시 준비가 필요하면 [Session Closeout](session-closeout.md)을 따른다.
